# Gwajae Implementation Roadmap

## 1. 개발 목표

최소 기능으로 빠르게 사용할 수 있는 과제 관리 웹사이트를 먼저 완성한다.

## 2. 권장 개발 순서

### Phase 1. 프로젝트 세팅

- React + TypeScript 프로젝트 초기화
- CSS Modules 구조 세팅
- 라우팅 구조 세팅
- 기본 레이아웃 및 탭 UI 구성

### Phase 2. Supabase 연동

- Supabase 프로젝트 생성
- Google OAuth 설정
- 환경변수 연결
- 로그인/로그아웃 플로우 연결

### Phase 3. 인증 및 권한

- 허용 이메일 검사 로직 구현
- 관리자/일반회원 역할 판별
- 보호 라우트 구현
- 관리자 전용 라우트 구현

### Phase 4. DB 및 Storage

- 테이블 생성
- 기본 과목 `미지정` 자동 생성 로직 구현
- Storage 버킷 생성
- 첨부 메타데이터 저장 로직 구현

### Phase 5. 과목 기능

- 과목 목록 조회
- 과목 생성
- 색상 선택
- 과목 삭제 시 `미지정`으로 이동

### Phase 6. 과제 기능

- 과제 목록 조회
- 마감일 기준 정렬
- 과제 생성 모달
- 즐겨찾기 토글
- 제출 여부 토글
- 과제 삭제

### Phase 7. 파일 업로드

- 이미지 여러 장 업로드
- 파일 여러 개 업로드
- 첫 이미지 썸네일 자동 지정
- 사용자별 업로드 용량 검사

### Phase 8. 홈 탭

- 개인 사용량 카드
- 관리자용 전체 사용량 카드
- 즐겨찾기 과제 목록
- 마감 임박 과제 목록

### Phase 9. 관리자 탭

- 허용 이메일 목록 조회
- 이메일 직접 추가
- 역할/활성 여부 표시

### Phase 10. 마감 정리

- 에러 처리
- 빈 상태 UI
- 로딩 상태 UI
- 업로드 실패 처리
- 모바일 레이아웃 점검

## 3. MVP 완료 기준

아래 조건을 만족하면 MVP 완료로 본다.

- Google 로그인 가능
- 허용되지 않은 이메일은 접근 불가
- 관리자만 관리자 탭 접근 가능
- 사용자가 과목을 만들 수 있음
- 사용자가 과제를 등록할 수 있음
- 사진과 파일을 첨부할 수 있음
- 즐겨찾기와 제출 여부를 변경할 수 있음
- 홈에서 사용량과 핵심 과제 정보를 볼 수 있음

## 4. 우선순위

### 최우선

- 로그인
- 권한
- 과목
- 과제 CRUD
- 파일 업로드

### 다음 우선순위

- 홈 대시보드 완성도
- 관리자 계정 관리 UX
- 빈 상태/에러 상태 개선

## 5. 리스크와 대응

### 리스크 1. 무료 스토리지 초과

- 대응:
  - 사용자별 `100MB` 제한 강제
  - 파일 업로드 전 예상 사용량 표시

### 리스크 2. 권한 설정 실수

- 대응:
  - RLS 우선 설계
  - 관리자도 타 사용자 과제는 못 보게 정책 분리

### 리스크 3. 과목 삭제 시 데이터 꼬임

- 대응:
  - 삭제 전에 `미지정` 과목 이동 로직 수행
  - 기본 과목은 항상 존재 보장

### 리스크 4. 이미지/파일 업로드 실패

- 대응:
  - 업로드 성공 후 DB 반영
  - 실패 시 사용자에게 개별 파일 상태 표시

## 6. 추천 초기 작업 목록

1. 프론트 프로젝트 생성
2. Supabase 프로젝트 생성
3. Google 로그인 설정
4. 환경변수 연결
5. 기본 탭 레이아웃 구현
6. DB 스키마 생성
7. 과제 목록/등록 UI 구현

## 7. 이후 확장 후보

- 과제 검색
- 과제 공유
- 썸네일 수동 지정
- 제출 완료 필터
- 파일 미리보기
- 관리자용 통계
