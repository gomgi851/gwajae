# Gwajae Product Spec

## 1. 프로젝트 개요

- 프로젝트명: `Gwajae`
- 목표: 친구들과 사용할 수 있는 과제 관리 웹사이트 구축
- 초기 대상 사용자:
  - 관리자 2명
  - 이후 초대된 일반 회원
- 핵심 컨셉:
  - 각 사용자는 자신의 과제만 관리
  - 관리자는 서비스 운영만 담당하고, 다른 사용자의 과제는 기본적으로 볼 수 없음
  - 직관적인 과제 등록, 첨부파일 업로드, 즐겨찾기, 제출 여부 관리 제공

## 2. MVP 범위

### 포함 기능

- Google 로그인
- 허용된 이메일만 로그인 후 서비스 이용 가능
- 관리자/일반회원 권한 분리
- 홈 탭
- 과제 관리 탭
- 관리자 탭
- 과목 생성/수정/삭제
- 과제 생성/조회/삭제
- 과제 즐겨찾기
- 과제 제출 여부 토글
- 사진 여러 장 업로드
- 파일 여러 개 업로드
- 사용자별 저장 용량 제한
- 관리자용 전체 스토리지 사용량 표시

### 제외 기능

- 사용자 간 과제 공유
- 댓글/채팅
- 푸시 알림
- 과제 상태 단계 관리(`미시작`, `진행중` 등)
- 관리자에 의한 타 사용자 과제 열람

## 3. 사용자 역할

### 관리자

- 허용 이메일 목록 관리 가능
- 일반 사용자와 동일하게 자신의 과제 관리 가능
- 관리자 탭 접근 가능
- 다른 사용자의 과제/파일은 기본적으로 조회 불가

### 일반 회원

- 자신의 과목/과제/파일만 관리 가능
- 관리자 탭 접근 불가

## 4. 정보 구조

### 탭 구성

1. 홈
2. 과제 관리
3. 관리자

## 5. 홈 탭

### 목적

- 사용자가 가장 먼저 자신의 상태를 확인하는 대시보드

### 구성 요소

- 내 저장공간 사용량 원형 그래프
  - 일반 사용자: `100MB` 기준 사용량 표시
  - 관리자: 개인 사용량 카드 + 전체 프로젝트 `1GB` 기준 사용량 카드 추가
- 즐겨찾기한 과제 목록
- 마감 임박 과제 목록

### 즐겨찾기 과제 목록 표시 항목

- 과목명
- 과제명
- 마감일
- 제출 여부

### 마감 임박 과제 목록 표시 기준

- 기본 정렬: 마감일 가까운 순
- 제출 여부와 무관하게 표시 가능
- 다만 UI에서는 제출 완료 상태를 구분 표시

## 6. 과제 관리 탭

### 목적

- 사용자의 모든 과제와 과목을 관리하는 메인 작업 공간

### 화면 구성

- 과목 필터 또는 과목 선택 UI
- 과목 추가 버튼
- 과제 등록 버튼
- 과제 표

### 과제 표 기본 컬럼

- 과목
- 과제명
- 마감일
- 제출 여부
- 즐겨찾기
- 링크
- 첨부 개수
- 삭제

### 기본 정렬

- 마감일 가까운 순 오름차순

### 과목 정책

- 과목은 단순 분류용
- 각 과목은 파스텔톤 프리셋 색상 중 하나를 선택
- 색상은 서비스가 제공하는 제한된 개수의 색상만 선택 가능
- 시스템 기본 과목 `미지정`은 항상 존재
- 과목 삭제 시, 해당 과목의 과제는 모두 `미지정` 과목으로 이동

### 과제 등록 방식

- 과제 등록 버튼 클릭 시 모달 오픈
- 모달에서 입력 후 저장

### 과제 등록 필드

#### 필수

- 과목
- 과제명
- 마감일
- 제출 여부

#### 선택

- 설명
- 링크
- 사진 여러 장
- 파일 여러 개

### 사진 정책

- 사진 업로드 개수 제한 없음
- 사진이 1장 이상이면 그 중 1장이 썸네일로 사용됨
- 초기 MVP에서는 첫 번째 업로드 이미지를 자동 썸네일로 지정

### 파일 정책

- 파일 업로드 개수 제한 없음
- 파일 형식은 일반적인 문서 업로드를 허용
- 실제 저장은 사용자 저장 용량 한도 내에서만 가능

### 제출 여부 정책

- 토글로 관리
- 제출 완료로 바꾸어도 목록에서 자동 삭제되지 않음
- 사용자가 직접 삭제한 경우에만 삭제됨

### 삭제 정책

- 과제 삭제는 사용자 명시적 동작으로만 수행
- 삭제 전 확인 UI 제공 권장

## 7. 관리자 탭

### 목적

- 허용된 사용자 계정 관리

### 접근 제한

- 관리자 계정만 접근 가능

### 제공 기능

- 허용 이메일 목록 조회
- 이메일 직접 추가
- 역할 확인(`admin`, `member`)
- 활성 여부 표시

### 이번 단계에서 포함하지 않는 기능

- 가입 승인 대기 화면
- 초대 메일 자동 발송
- 사용자 활동 로그

## 8. 인증 및 접근 정책

### 로그인 방식

- Google OAuth 로그인

### 서비스 접근 방식

1. 사용자가 Google 계정으로 로그인
2. 로그인된 이메일이 허용 목록에 있는지 확인
3. 허용된 이메일만 서비스 이용 가능

### 접근 제어 원칙

- 로그인 허용 판단: 이메일 기반 whitelist
- 데이터 접근 제어: 사용자 고유 ID 기반
- 관리자는 운영 권한만 가지며, 타 사용자 데이터 열람 권한은 기본 제공하지 않음

## 9. 저장공간 정책

### 사용자 기준

- 각 사용자는 최대 `100MB`까지 업로드 가능

### 관리자 대시보드 기준

- 전체 프로젝트 스토리지 사용량을 `1GB` 기준으로 확인 가능

### 표시 정책

- 홈 탭에서 사용량을 원형 그래프로 시각화
- 수치도 함께 노출하는 것을 권장
  - 예: `34MB / 100MB`

## 10. UX 원칙

- 과제 관리 화면은 카드형보다 표형 중심
- 주요 액션은 빠르게 접근 가능해야 함
- 사용자가 현재 마감 우선순위를 한눈에 볼 수 있어야 함
- 디자인은 프롬프트 기반 생성 도구를 활용하되, 정보 구조는 본 문서 기준으로 유지

## 11. 향후 확장 아이디어

- 과제 공유 기능
- 과제 복제 기능
- 과제 검색
- 과목별 정렬/필터 강화
- 썸네일 수동 변경
- 제출 완료 과제만 모아보기
- 알림 기능
